Análisis de las Diferencias:

  • Ground Truth (9 nodos, 7 aristas): La arquitectura oficial de la publicación FAST25 no divide EventBridge ni Glue en cajas múltiples. Es un diseño simple y lineal: SQL Server (ThirdParty) → DMS → S3 → SQS → Glue → S3 (Delta Lake) → User. EventBridge → StepFunctions → Glue actúan en la parte superior.
  • Gemini Standard (9 nodos, 9 aristas): Logra la misma estructura lineal que el Ground Truth. Sin embargo, mapeó de forma imprecisa la base de datos SQL Server origen como un nodo OnPremDC en lugar de ThirdParty. Además, alucinó algunas conexiones redundantes de retorno en las aristas.
  • Gemini Parsimonioso (9 nodos, 8 aristas): Corrige la base de datos SQL Server mapeándola a ThirdParty, alineándose de forma exacta con la publicación. La estructura de aristas es sumamente limpia y reduce las aristas alucinadas, logrando casi una equivalencia exacta con el Ground Truth.
